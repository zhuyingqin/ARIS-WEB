# 共享输出契约

本仓库所有会写文件的技能，默认都遵循同一套输出契约。

## 默认只生成 3 个核心文件

- `manifest.json`
  - 记录任务参数、统计信息、输出路径和契约版本。
- `records.json`
  - 记录该技能最核心的结构化结果。
- `summary.md`
  - 记录给人看的摘要、结论和下一步建议。

## 额外文件不再默认生成

如果某个技能还需要提供便于人工查看或外部工具导入的附加文件，例如：

- 候选级 Markdown 报告
- RIS
- 纯文本 chunk
- 标题/摘要拆分文本

则统一要求：

1. 只在显式传入 `--write-extras` 时生成。
2. 统一写到 `extras/` 目录。
3. 不再把这些文件平铺在输出根目录。

## 设计目标

- 让所有技能的最终目录结构更容易预测。
- 降低默认文件数量，避免根目录堆满技能私有产物。
- 既保留机器可读接口，也保留按需扩展的人类便利用文件。
